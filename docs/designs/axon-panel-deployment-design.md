## Overview

This document defines the standalone deployment architecture for AxonPanel, a web-based debug tool for Axon recorder. The design provides a simple self-contained executable with an embedded HTTP server for development and testing scenarios.

### Current State

AxonPanel is a Vue 3 single-page application (SPA) located at `apps/axon_panel/`. Currently:
- Frontend assets are built with Vite (`npm run build` → `dist/`)
- Requires external web server (nginx/caddy) to serve static files
- Connects to `axon_recorder` HTTP RPC API at configurable endpoint

### Target State

Standalone deployment mode provides:
- Self-contained executable with embedded web server
- Bundled static assets (HTML/CSS/JS) within the binary
- Single-command launch: `axon_panel --port 8082`
- No external dependencies for serving the web interface

**Port Convention:**
- axon_recorder HTTP RPC: 8080 (default)
- axon_recorder WebSocket: 8081 (HTTP RPC port + 1)
- axon_panel: 8082 (HTTP RPC port + 2)

## Requirements

### Functional Requirements

1. **Embedded Web Server**
   - Serve AxonPanel frontend assets (HTML, CSS, JavaScript)
   - Handle SPA routing (fallback to index.html for client-side routes)

2. **Asset Embedding**
   - Bundle all frontend build artifacts into the executable
   - Zero external file dependencies for serving the UI

3. **Simple Configuration**
   - CLI flags for port and recorder endpoint
   - Sensible defaults for quick startup

### Non-Functional Requirements

1. **Simplicity**
   - Minimal code and dependencies
   - Easy to build and deploy
   - Quick startup time

2. **Reliability**
   - Graceful shutdown on SIGTERM/SIGINT

```
│  axon_panel (Standalone Binary)                         │
│                                                         │
│  ┌────────────────────────────────────────────────────┐ │
│  │  Embedded HTTP Server (C++ / cpp-httplib)          │ │
│  │  - Static file serving                             │ │
│  │  - SPA routing fallback                            │ │
│  └────────────────────────────────────────────────────┘ │
│                                                         │
│  ┌────────────────────────────────────────────────────┐ │
│  │  Embedded Assets (xxd-generated byte arrays)       │ │
│  │  - index.html                                      │ │
│  │  - JavaScript bundles                              │ │
│  │  - CSS stylesheets                                 │ │
│  └────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
                          │
                          │ HTTP
                          ▼
                ┌─────────────────────┐
                │  axon_recorder      │
                │  HTTP RPC: 8080     │
                │  WebSocket: 8081    │
                └─────────────────────┘
```

### Deployment Mode

```
User Browser → axon_panel:8082 → axon_recorder:8080 (HTTP RPC)
               (serves UI)                      :8081 (WebSocket)
```

- AxonPanel runs as independent process
- Serves web UI on dedicated port (default: 8082, recorder port + 2)
- Frontend makes direct API calls to recorder HTTP RPC (8080) and WebSocket (8081)

## Technology Choice: C++ + cpp-httplib

C++ provides the best fit for this project:
- **Consistency**: Same language as axon_recorder, no new toolchain
- **Simplicity**: Header-only library (cpp-httplib) is easy to integrate
- **Performance**: Native performance with minimal overhead
- **Build system**: Seamless CMake integration

**Asset Embedding Strategy:**
Use `xxd -i` to convert frontend build artifacts into C++ byte arrays at build time.

**Simple Implementation:**
```cpp
#include "httplib.h"
#include "embedded_assets.h"

int main() {
    httplib::Server svr;

    svr.Get("/.*", [](const httplib::Request& req, httplib::Response& res) {
        std::string path = req.path == "/" ? "/index.html" : req.path;
        auto asset = get_embedded_asset(path);

        if (asset) {
            res.set_content(asset->data, asset->size, asset->mime_type);
        } else {
            // SPA fallback
            auto index = get_embedded_asset("/index.html");
            res.set_content(index->data, index->size, "text/html");
        }
    });

    svr.listen("0.0.0.0", 8082);
}
```

### Build Process

```cmake
# apps/axon_panel/CMakeLists.txt

# Build frontend assets
add_custom_command(
    OUTPUT ${CMAKE_CURRENT_BINARY_DIR}/frontend_dist.stamp
    COMMAND npm run build
    WORKING_DIRECTORY ${CMAKE_CURRENT_SOURCE_DIR}/frontend
    COMMENT "Building frontend assets"
)

# Generate embedded asset headers using xxd
add_custom_command(
    OUTPUT ${CMAKE_CURRENT_BINARY_DIR}/embedded_assets.cpp
    COMMAND ${CMAKE_CURRENT_SOURCE_DIR}/scripts/embed_assets.sh
            ${CMAKE_CURRENT_SOURCE_DIR}/frontend/dist
            ${CMAKE_CURRENT_BINARY_DIR}/embedded_assets.cpp
    DEPENDS ${CMAKE_CURRENT_BINARY_DIR}/frontend_dist.stamp
    COMMENT "Embedding frontend assets"
)

# Build axon_panel binary
add_executable(axon_panel
    server/main.cpp
    ${CMAKE_CURRENT_BINARY_DIR}/embedded_assets.cpp
)

target_link_libraries(axon_panel PRIVATE httplib::httplib)
```

### Configuration

Simple CLI arguments only:

```bash

axon_panel

```

No YAML config files needed for a debug tool.

**Note:** Default port 8082 follows convention: recorder HTTP RPC port (8080) + 2.

### Server Implementation

```cpp
// server/main.cpp
#include "httplib.h"
#include "embedded_assets.hpp"
#include <iostream>

int main(int argc, char* argv[]) {
    int port = 8082;  // Default: recorder HTTP RPC port (8080) + 2

    // Simple argument parsing
    for (int i = 1; i < argc; i++) {
        if (std::string(argv[i]) == "--port" && i + 1 < argc) {
            port = std::stoi(argv[++i]);
        }
    }

    httplib::Server svr;

    // Serve embedded assets
    svr.Get("/.*", [](const httplib::Request& req, httplib::Response& res) {
        std::string path = req.path == "/" ? "/index.html" : req.path;
        auto asset = EmbeddedAssets::get(path);

        if (asset) {
            res.set_content(asset->data, asset->size, asset->mime_type.c_str());
        } else {
            // SPA fallback
            auto index = EmbeddedAssets::get("/index.html");
            res.set_content(index->data, index->size, "text/html");
        }
    });

    std::cout << "AxonPanel running on http://0.0.0.0:" << port << std::endl;
    svr.listen("0.0.0.0", port);
    return 0;
}
```

## Security Considerations

As a debug tool, AxonPanel has minimal security requirements:

1. **Network Binding**
   - Binds to 0.0.0.0 by default for easy access
   - Users should configure firewall rules if needed

2. **No Authentication**
   - Not required for debug/development scenarios
   - Rely on network isolation

3. **CORS**
   - Recorder must enable CORS for panel access
   - Simple configuration in recorder

## Testing Strategy

### Basic Tests

1. **Server Startup**
   - Verify server starts on configured port
   - Responds to HTTP requests

2. **Asset Serving**
   - All frontend assets accessible
   - SPA routing works (fallback to index.html)

3. **Integration**
   - Panel connects to recorder API
   - UI controls work end-to-end

## Risks and Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Asset embedding complexity | Low | Use simple xxd tool |
| Binary size increase | Low | Acceptable for debug tool |
| CORS configuration | Low | Document in recorder setup |
