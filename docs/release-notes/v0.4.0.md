# Axon v0.4.0 Release Notes

Axon v0.4.0 expands the recorder into a more complete edge operations stack. The release keeps the high-throughput MCAP recorder at the center, adds sidecar-optional metadata and MCAP-only transfer, and introduces local agent/system services for robot process control and observability.

## Highlights

- **Sidecar-optional recording**: MCAP metadata now carries the task, device, recording, and topic summary records needed by downstream systems even when JSON sidecar generation is disabled.
- **MCAP-only upload path**: `axon-transfer` can use `<task_id>.mcap.done` as the completion signal with `scanner.require_json_sidecar: false`.
- **Robot configuration workspace**: AxonPanel can browse, upload, validate, scan, enable/disable, diff, and review robot configuration files.
- **Fleet operations services**: `axon-agent` manages robot profiles, action manifests, adapter plugins, and supervised processes; `axon-system` reports process/resource status and alerts.
- **Recorder diagnostics**: `/rpc/status`, callbacks, and WebSocket updates include disk guard state, sidecar state, Keystone time-gap diagnostics, task snapshots, and optional incident debug bundle details.
- **Release packaging**: Debian metadata, package fallbacks, standalone build scripts, and systemd units are aligned to `0.4.0`.

## Migration Notes

- Existing MCAP+JSON deployments can keep `scanner.require_json_sidecar: true`; this remains the compatibility mode for transfer configs.
- MCAP-only deployments should disable recorder sidecar generation and set `scanner.require_json_sidecar: false` in transfer configs. The recorder must write a fresh `<task_id>.mcap.done` marker after finalization.
- Downstream indexers that previously depended on `<task_id>.json` should read `axon.task`, `axon.device`, `axon.recording`, and `axon.topics` MCAP metadata before disabling sidecars fleet-wide.
- `sidecar_path: null` in callbacks or `/rpc/status` now means "no JSON sidecar was expected or generated"; clients should not treat it as an error by itself.
- The new `axon-agent` and `axon-system` packages install separate systemd services. Review enablement policy before adding them to production robot images.

## Validation Focus

- Build/install standalone Debian packages for the target Ubuntu/ROS distribution.
- Verify recorder finalization in both sidecar-enabled and sidecar-disabled modes.
- Verify transfer upload and ACK cleanup in both `require_json_sidecar: true` and `false` modes.
- Verify AxonPanel config workspace against the intended `AXON_PANEL_CONFIG_DIR`.
- Verify `axon-system --version`, `axon-agent` service startup, and package service unit installation.
